축약 = supernetting   여러 네트워크를 하나로 축약 = subnetting의 반대 개념


<예>
192.168.10.1
192.168.20.1
192.168.30.1
-----------------------------------------------------------
서브넷마스크             네트워크 주소
255.255.255.0 -> (192.168.10.0, 192.168.20.0, 192.168.30.0)
255.255.0.0   -> (192.168.0.0)
255.0.0.0     -> (192.0.0.0)

<예>     축약의 원리 ==> 비교해서 같은 비트는 1, 다른 비트는 0
192.168.00000 001.00000000
192.168.00000 010.00000000
192.168.00000 011.00000000
192.168.00000 100.00000000
192.168.00000 101.00000000
-----------------------------------------
255.255.11111 000.00000000 = 255.255.248.0 --> 다르면 그 시점부터 무조건 0 계속
192.168.00000 000.00000000 = 192.168.0.0

<예>
192.168.10(00001 010).0
192.168.11(00001 011).0
192.168.12(00001 100).0
192.168.13(00001 101).0
192.168.14(00001 110).0
192.168.15(00001 111).0
------------------------------------------
축약된 Subnet Mask = 255.255.248.0
축약된 Network 주소 = 192.168.8.0 --> 서브넷 마스크에서 1로 표시된 부분은 유지, 나머지 
 0으로 표시된 부분을 다 없애고 0으로 취급
192.168.00001 010.00000000 --> 합쳐야할 네트워크 주소 중 하나
255.255.11111 000.00000000
192.168.00001 000.00000000 --> 합쳐진 결과물
            |--> 이것까진 네트워크 주소로 그대로 고정. 이거 뒤부터 무조건 0으로 바꿔

<예> 
211.175.185(10111 001).0
211.175.186(10111 010).0
211.175.187(10111 011).0
211.175.188(10111 100).0
-----------------------------------------
축약된 Subnet Mask = 255.255.248.0        11111000
축약된 Network 주소 = 211.175.184.0
